## 功能概述

该文件实现了伙伴（Companion）的生成逻辑。伙伴的外观、稀有度、属性等"骨骼"部分完全基于用户ID哈希值确定性生成，确保相同用户始终获得相同的伙伴，且无法通过编辑配置来伪造。

使用Mulberry32种子化伪随机数生成器（PRNG）和FNV-1a哈希算法，保证跨平台一致性。

## 核心内容详解

### 导入与依赖

| 依赖 | 来源 | 用途 |
|------|------|------|
| `getGlobalConfig` | `../utils/config.js` | 获取全局配置 |

### 核心算法

#### `mulberry32` - 种子化PRNG
- **算法**: Mulberry32（轻量级种子化随机数生成器）
- **输入**: 32位整数种子
- **输出**: [0, 1) 范围内的伪随机数
- **特点**: 确定性、快速、分布良好

#### `hashString` - 字符串哈希
- **Bun环境**: 使用 `Bun.hash()` 并取低32位
- **其他环境**: FNV-1a算法（32位）
- **用途**: 将用户ID转换为32位整数种子

### 生成函数

#### `rollRarity` - 稀有度生成
- 基于权重表 `RARITY_WEIGHTS` 进行加权随机
- 概率分布：common(60%)、uncommon(25%)、rare(10%)、epic(4%)、legendary(1%)

#### `rollStats` - 属性生成
- **保底机制**: 根据稀有度确定属性最低值
  - common: 5, uncommon: 15, rare: 25, epic: 35, legendary: 50
- **峰值属性**: 随机选择一项属性作为强项（+50~80）
- **弱势属性**: 随机选择一项不同属性作为弱项（保底-10~+5）
- **其他属性**: 保底~保底+40范围内随机

#### `rollFrom` - 完整生成
生成包含以下内容的 `Roll` 对象：
- `bones`: 完整伙伴骨骼
  - `rarity`: 稀有度
  - `species`: 物种（18选1）
  - `eye`: 眼睛样式（6选1）
  - `hat`: 帽子（稀有度为common时强制'none'，否则8选1）
  - `shiny`: 闪光效果（1%概率）
  - `stats`: 五项属性
- `inspirationSeed`: 灵感种子（用于模型生成名字和个性）

### 缓存机制

#### `rollCache` - 单槽缓存
- **目的**: 优化热点路径（500ms精灵刷新、每次按键、每轮观察）
- **键**: `userId + SALT`（盐值：'friend-2026-401'）
- **策略**: 单条目缓存，相同key直接返回缓存值

### 导出函数

| 函数 | 用途 |
|------|------|
| `roll(userId)` | 基于用户ID生成伙伴（带缓存） |
| `rollWithSeed(seed)` | 基于任意种子生成伙伴 |
| `companionUserId()` | 获取当前用户ID（OAuth accountUuid → userID → 'anon'） |
| `getCompanion()` | 获取完整伙伴对象（骨骼+灵魂），未孵化返回undefined |

### 数据合并逻辑

`getCompanion()` 返回的对象结构：
```
{ ...storedSoul, ...bones, hatchedAt }
```

骨骼字段在后，确保旧格式配置中的过时骨骼字段被覆盖。

## 设计要点

1. **确定性**: 相同输入始终产生相同输出
2. **防篡改**: 骨骼实时生成，不存储，防止伪造
3. **性能**: 单槽缓存优化高频调用
4. **跨平台**: Bun hash优先，降级FNV-1a

## 注意事项

1. **盐值变化**: 修改 `SALT` 会导致所有用户伙伴改变
2. **缓存失效**: 单槽缓存仅在key变化时更新，内存占用极小
3. **降级路径**: 非Bun环境使用FNV-1a，结果可能与Bun不同（但实际中用户固定平台）
